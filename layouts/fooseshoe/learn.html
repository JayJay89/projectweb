Review

I need to find a way to combat inline-block gaps, the current one is setting a gapfix: 0px. This requires putting font size on all its child. Using font-size: initial on its childrens doesn't work as well

While designing navbars, Do I use line-height for vertical size control? and vertical-align: middle. For its side, do I use padding left and right?

While designing a system with mixins, note that sass doesn't support calculation between % and px. But there's a fix for that! use native css calc() function

While designing .shoe-name, i forgot to take into account that text might be broken down into two lines, and that alter the design of the layout. In this case Buffalo Decollete was broken down into two lines

While designing the dropdown for hamburger, I need to think about the design before working on the HTML. The HTML need to work on both normal and small size. Think about the design first before laying down the HTML frame

Inputs does not support Pseudo Elements and hence the searchbar icon cannot be applied on the right side of the bar that way.

I need to set a general padding space, so that I will not randomly set padding spaces whenever I need them

I also need to identify the elements which are shared, which are repeated multiple times, and plan them accordingly

Image with max-width: 100% will be resizeable. Ideally use min-width and max-width

I need to find a better solution to equal height design. The current one employs border to simulate gap between the boxes

Using class names such as .col-3, .col-6 isn't helpful. These classes are given widths like 33.3% and 66.6%. However, at smaller size screens, they will be given 100% width. Hence, the class name isn't semantically correct

All given class name should be functionally related.
